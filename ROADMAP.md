# つんどけんさく ロードマップ

最終更新: 2026-07-12

## コンセプト

- 積読を知識に変える
- AI（NotebookLM や ChatGPT など）へ渡す資料を組み立てる、ローカル完結型ワークスペース

つんどけんさくは AI 要約・ベクトル検索・RAG を内蔵しません（[README「あえて入れていないもの」](README.md#あえて入れていないもの) 参照）。AI はあくまで外部のサービスとして使い、このアプリは **AI に渡す前（資料の組み立て）と、AI から戻った後（知識の定着）** を支えることに徹します。

利用者の基本の流れは「**検索 → 資料へ追加 → 並び替え → エクスポート**」です。ロードマップはこの体験を先に完成させ、そのうえで AI 向けの最適化・帰還へ進みます。

## 現在地

### Phase 0: 検索基盤 — 完了

「蔵書から探す」は一段落しました。

- 蔵書横断検索: PDF 本文・タイトル・Kindle 本・Scrapbox/Cosense メモ・書籍ノート（FTS5 + Sudachi 分かち書き + trigram 部分一致）
- 検索条件: AND/OR 切替（「すべての語を含む」チェックボックス）、`"フレーズ検索"`、`-除外語` / `-"フレーズ除外"`
- タイトル推定: PDF メタデータ → Scrapbox → ファイル名整形のフォールバック、OCR ノイズ除外、`refresh-titles` コマンド

### Phase 1: 資料棚・資料の永続化 — 完了（2026-07-07）

ワークスペース改め**資料棚**で、「問い（テーマ）」単位の**名前付き資料**（内部名はパック）を SQLite に永続化しました。かつての sessionStorage 揮発カートは廃止し、初回アクセス時に自動で資料へ移行されます。

- 資料の作成・切替・改名・削除（資料棚のセレクトボックス）
- 初回は資料0件から始まり、必要になったタイミングで作る（自動作成なし）
- 出典情報の基礎（追加時点のタイトル・ページ範囲・追加日時）を保存 — 後続 Phase のデータ基盤

### Phase 2: 資料を組み立てる体験 — 進行中

Phase 2 の目的は、検索結果を集めることではなく、**AI へ渡せる資料として完成させる体験**を提供することです。検索結果を集め、並び替え、PDF / Markdown として出力するまでの一連を迷いなく完了できる状態にします。AI 向け最適化より先にこの体験を完成させます。操作性は完成後も実際に使いながら継続的に改善します。

- [x] 資料へ追加 — 検索結果・PDF プレビュー（検索起点・資料棚起点の両方）・本文検索モーダル・アウトライン章選択から。資料がなければ作成 → 切替 → 追加まで一連で実行
- [x] 削除 — 本単位の削除・資料を空にする・資料ごと削除
- [x] 並び替え — 書籍カードの「上へ」「下へ」（エクスポート順に反映）
- [x] PDF / Markdown エクスポート — 資料の並び順どおりに本ごと出力
- [x] 追加先資料の明示 — 検索結果下部のバーに「追加先: 資料名（n冊）」を常時表示。プルダウンで切替・その場で新規作成もできる。PDF プレビューの追加ボタンと成功メッセージにも資料名を表示
- [x] PDF プレビューの操作整理 — ツールバーを役割別2段（上段=見る操作、下段=集める操作）に再構成。「資料に追加」を主役に、切出系はドロップダウンへ集約
- [x] 検索結果のまとめ表示をデフォルトに — 「同じ書籍をまとめる」を初期 ON にし、1冊1カードでヒットページを一括追加できる状態を標準にする
- [x] 資料作成の操作性のブラッシュアップ第1弾 — 検索画面の「全てクリア」撤去、検索起点 PDF プレビューからの追加、資料を空にした後の冊数表示同期、ホーム画面への資料カード追加、ZIP一括エクスポート、ページ範囲編集のチップ表示化、PDFモーダルのサムネイルページ選択UI、サムネイル画質の改善、同一PDFを複数の資料項目として扱う機能（設計: [docs/pack-item-identity-design.md](docs/pack-item-identity-design.md)）
- [ ] 資料棚の情報設計・操作階層を整理（ユーザーストーリーマッピングなどで主要操作経路を確認しながら進める）
  - [ ] 資料の選択・組み立て・書き出し・管理操作をグループ化し、日常操作とバックアップ・削除などの低頻度操作を分離する
  - [ ] 用語と統計表示を整理（資料数・冊数・項目数・ページ数を一貫した表現に統一）
  - [ ] ページ追加・ホーム画面の導線を実利用を踏まえて改善する
    - [ ] 「ページを追加」を中心に、PDF・本文検索・章選択それぞれの追加方法の分かりにくさを洗い出す
    - [ ] ホーム画面と資料棚画面の役割を整理し、ホーム画面から主要操作へ直接アクセスできるようにする
- [ ] PDFページ選択UIの操作性・一覧性を改善 — サムネイルの小型化、拡大アイコンのホバー時表示、前後10ページ移動をモーダル上下双方から行えるようにする、を一体で対応する

## ロードマップ

### Phase 3: AI 向けエクスポート最適化 — 進行中

**利用者の到達状態**: 資料棚で組み立てた内容を作り直したり、外部 AI の制限に合わせて手作業で分けたりせず、宛先に適した形式・分量・出典を保ったまま書き出せる。

設計: [docs/ai-export-optimization-design.md](docs/ai-export-optimization-design.md) / 実装前監査: [docs/phase3c-3d-design-review.md](docs/phase3c-3d-design-review.md)

- [x] **Phase 3A: 資料の規模を事前に把握できる** — 完了（2026-07-11）
  - **利用者の到達状態**: AIへ渡す前に、資料の冊数・項目数・ページ数・おおよそのトークン数を確認できる。
  - **実装**: トークンバジェット可視化

- [x] **Phase 3B: AIに合わせた書き出し方法を選べる** — 完了（2026-07-11）
  - **利用者の到達状態**: 渡すAIに応じて最適な書き出し方法を選択できる。
  - **実装**: エクスポートプロファイル基盤

- [x] **Phase 3C: ChatGPT / Claudeへそのまま渡せる** — 完了（2026-07-12）
  - **利用者の到達状態**: 資料を作り直すことなく、ChatGPTやClaudeへそのまま渡せる形で書き出せる。
  - **実装**: Markdown分冊、出典保持、ZIPダウンロード、エクスポート履歴
  - ユーザーストーリーと設計: [docs/ai-export-optimization-design.md](docs/ai-export-optimization-design.md)
  - 履歴の確定仕様: [docs/export-events-design.md](docs/export-events-design.md)

- [ ] **Phase 3D: NotebookLMへそのまま渡せる**
  - **利用者の到達状態**: NotebookLMへ読み込ませやすいPDFとして書き出せる。
  - **実装**: 隣接項目のPDF結合、ページ上限・ソース数への対応

- [ ] **Phase 3E: 安心して使える品質に仕上げる**
  - **利用者の到達状態**: 書き出し結果が安定し、実運用で安心して利用できる。
  - **実装**: トークン係数調整、Playwright E2E、ドキュメント更新、AI向け書き出し導線の統合（資料完成後の主要操作としての明示、ホーム画面からの開始）

既存のエクスポート機能を発展させ、資料棚からAIへの受け渡しを完成させます。

### Phase 4: AI 成果物の帰還

**利用者の到達状態**: AIとの対話で得た知識を資料へ戻し、あとから検索・再利用できる。

- AIの回答・ノートを取り込み、生成元の資料と紐づけて保存
- 帰還した知識を蔵書横断検索の対象にする

現状では AI とのやりとりで得た知見が AI サービス側に置き去りになります。この段階で「積読 → 資料 → AI → 知識」のループが初めて閉じ、コンセプトの「知識に変える」が完結します。

前提: 成果物と生成元資料の紐づけには「エクスポート時点の構成スナップショット」が必要で、Phase 3C で開始するエクスポート履歴（[確定仕様](docs/export-events-design.md)）がその基盤になり得ます。事前調査: [docs/ai-artifact-return-discovery.md](docs/ai-artifact-return-discovery.md)

### Phase 5: 消化マップ・対比・共起

**利用者の到達状態**: 積読の活用状況や本同士の関係が見え、次に読む・比較する・深掘りする資料を見つけやすくなる。

- 消化マップ
- 対比資料
- 共起提示

いずれも利用履歴が効いてくる段階です。ただし現状は資料の削除・変更で過去の構成が消えます。Phase 3C から記録するエクスポート実績は、将来、消化マップ・共起のシグナルとして利用できる可能性があります（記録開始前の実績は復元不能）。事前調査: [docs/usage-history-discovery.md](docs/usage-history-discovery.md)

## やらないこと

- AI 要約・ベクトル検索・RAG の内蔵（AI は外部サービスに任せる）
- 一般的な PDF ビューア・ファイル管理ソフトの機能の追いかけ
